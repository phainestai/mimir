# Activity: Revise Conflict Map and Build Manifest

**Activity ID**: 108
**Order**: 5
**Phase**: Planning
**Dependencies**: Predecessor: Activity 107 (Run BPE-01 and Create Skeletons per Scenario)

## Description

Revise Conflict Map and Build Manifest

## Guidance

## Purpose
Replace the rough area-level conflict map from PIT-03 with a precise file-level conflict map derived from skeleton commits. Build the machine-readable execution manifest YAML that MIT will consume as its ground truth.

## Prerequisites
- All scenario skeleton commits from PIT-04
- Rough parallel groups from PIT-03

## Steps

### Step 1: Collect Exact File Footprints
For each scenario skeleton commit, extract the exact file list:
```bash
# Find skeleton commits by message pattern
git log --oneline --format="%H %s" | grep "chore(pit): skeleton"

# For each commit hash, get files touched
git diff {commit_hash}~1 {commit_hash} --name-only
```

Build a table:
| Scenario | Files |
|---|---|
| S1 | mcp_integration/tools.py, methodology/services/workflow_export_service.py, tests/integration/test_mcp_export.py |
| S2 | mcp_integration/tools.py, methodology/services/workflow_import_service.py, tests/integration/test_mcp_import.py |
| S3 | methodology/views/agent_views.py, methodology/templates/agents/list.html, tests/integration/test_agent_views.py |

### Step 2: Rebuild Conflict Matrix
For each pair of scenarios, check if they share any file:
```
S1 ∩ S2 = {mcp_integration/tools.py}  → CONFLICT → serialize
S1 ∩ S3 = {}  → no conflict → can parallel
S2 ∩ S3 = {}  → no conflict → can parallel
```

### Step 3: Adjust Parallel Groups
If new conflicts found that weren’t in PIT-03 rough map: serialize those scenarios.
If PIT-03 over-serialized (conflict suspected but not real): can now parallelize.

Final groups:
```
Group A: [S1]
Group B: [S3]   ← parallel with A (no shared files)
Group C: [S2]   ← depends on S1 (shared tools.py)
```

### Step 4: Write Execution Manifest YAML
Create file at `docs/plans/iterations/ITER-YYYYMMDD-HHmm-goal-slug.yaml`:

```yaml

# Edit source: GitHub Milestone #{N} description

  doctrine_version: "1.0"
  created_at: "2026-04-13T18:00:00Z"

conflict_map:
  mcp_integration/tools.py: [S1, S2]

parallel_groups:
  A: [S1]
  B: [S3]
  C: [S2]

scenarios:
  S1:
    title: "User can export workflow as JSON"
    github_issue: 142
    parallel_group: A
    time_budget_minutes: 55
    codebase_footprint:
      - mcp_integration/tools.py
      - methodology/services/workflow_export_service.py
      - tests/integration/test_mcp_export.py
    checkpoint:
      command: "pytest tests/integration/test_mcp_export.py -x"
      expected_exit_code: 0
    dependencies: []
    rollback_point: "git stash"
  S2:
    title: "User can import workflow from JSON"
    github_issue: 143
    parallel_group: C

    codebase_footprint:
      - mcp_integration/tools.py
      - methodology/services/workflow_import_service.py
      - tests/integration/test_mcp_import.py
    checkpoint:
      command: "pytest tests/integration/test_mcp_import.py -x"
      expected_exit_code: 0
    dependencies: [S1]
    rollback_point: "git stash"
  S3:
    title: "User can manage Agents via web UI"
    github_issue: 144
    parallel_group: B
    time_budget_minutes: 45
    codebase_footprint:
      - methodology/views/agent_views.py
      - methodology/templates/agents/list.html
      - tests/integration/test_agent_views.py
    checkpoint:
      command: "pytest tests/integration/test_agent_views.py -x"
      expected_exit_code: 0
    dependencies: []
    rollback_point: "git stash"
```

### Step 5: Commit Manifest
```bash
git add docs/plans/iterations/
git commit -m "chore(pit): execution manifest ITER-YYYYMMDD-HHmm-goal-slug"
```

## Success Criteria
